# Автоматическая проверка программного кода студентов

## Цель проекта 

Настроить автоматическую проверку программ студентов и отправку обратной связи в личном кабинете.


## Обоснование проекта

У студентов и учеников, изучающих **программирование** (особенно основы какого-либо языка), в личном кабинете есть задания на написание программного кода. Эти задания нужны, чтобы студенты могли: 
- Закреплять пройденные знания
- Учиться писать код самостоятельно 
- Вырабатывать алгоритмическое мышление

Для изучения более сложных тем нужна **очень крепкая база**, поэтому заданий на отработку должно быть **достаточное количество**. Сейчас заданий на написание кода в личном кабинете **не очень много**, при этом студенты всегда вынуждены **ждать**, когда преподаватель проверит их решения. Это занимает **время**, некоторые преподаватели не проверяют ДЗ днями, а иногда – неделями. Из-за этого мотивация студентов решать такие задания постепенно падает. Как результат – студенты **не закрепляют важные базовые навыки**, из-за чего в дальнейшем:
- Не могут качественно изучать более сложные темы 
- Не смогут научиться сложным практическим навыкам

Это напрямую влияет на **качество обучения** и **ожидания студентов**. К тому же, на большинстве образовательных платформ, где есть изучение программирования, почти всегда задания на написание программного кода проверяются не преподавателем, а в **автоматическом формате**. Это касается даже популярных бесплатных курсов, например «Поколение Python: курс для начинающих»: https://stepik.org/course/58852/info. 

Таким образом, создание системы с автоматической проверкой программного кода поможет обеспечить следующее:
- У студентов будет необходимое количество заданий для отработки навыков написания кода и развития алгоритмического мышления
- Студенты смогут практически моментально получать обратную связь по написанным программам
- У студентов будет больше мотивации выполнять ДЗ, как следствие – студенты будут лучше закреплять пройденный материал и быстрее усваивать новые, более сложные знания

### Общая схема работы автоматической системы проверок

На данный момент путь ученика при работе с ДЗ, где нужно писать программный код, выглядит следующий образом:

1. Студент заходит в личный кабинет
2. Студент выбирает нужный курс, где есть нерешенное ДЗ
3. Студент выбирает тему и открывает список со всеми заданиями
4. Студент открывает задание на написание программного кода
5. Студент читает задание и пишет программный код с решением
6. Студент отправляет написанный программный код на проверку преподавателю
7. Преподаватель заходит в личный кабинет
8. Преподаватель открывает задание на проверку программного кода
9. Преподаватель читает задание, проверяет код студента, выставляет балл за задание и пишет комментарий 
10. Преподаватель отправляет результат проверки обратно студенту
11. Если решение было неверным, то преподаватель может вернуть задание студенту на доработку
12. Если решение было верным, то студент в личном кабинете видит положительную обратную связь от преподавателя

С учетом наличия системы **автоматической проверки** путь ученика при работе с ДЗ может выглядеть так:

1. Студент заходит в личный кабинет
2. Студент выбирает нужный курс, где есть нерешенное ДЗ
3. Студент выбирает тему и открывает список со всеми заданиями
4. Студент открывает задание на написание программного кода
5. Студент читает задание и пишет программный код с решением
6. Студент отправляет написанный программный код на проверку системе
7. **Система в течение нескольких секунд проверяет решение студента**
8. Если решение было верным, то студент видит сообщение об успешном выполнении задания
9. Если решение было неверным, то студент видит сообщение об ошибке в решении и может попробовать решить задание еще раз

Видно, что благодаря системе время проверки **сокращается с нескольких дней** (в среднем) до **нескольких секунд** (гарантированно).
